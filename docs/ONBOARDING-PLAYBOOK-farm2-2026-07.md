# ONBOARDING PLAYBOOK — farm #2 and beyond (white-glove, productized) (2026-07)

*The paid-onboarding offer as a fixed-scope checklist, so each new farm costs predictable founder-hours and the hand-off (DFY→DIWY) is designed in from day one. Grounded in: GTM research (services-led verdict + 4 hard rules), customer research (barriers: literacy → training works +20pp; trust; messy ground truth), ops research (the paper artifacts to mirror). Cap: 2–3 concurrent onboardings. Owner: Amr Ebeid.*

## The offer (what the farm buys)

**«إعداد وتشغيل نظام المزرعة» — fixed scope, fixed fee** (fee = the headline number, repriced each season): we set up the farm's structure, registry, store, people, and season program; train the manager + storekeeper + accountant; run the first 2 weeks alongside them; hand over with a trained farm clerk. Subscription starts at go-live.

## Phase 0 — Qualify + contract (Owner, ~2h)
- [ ] Fit check: ≥1,000 palms OR export ambition OR multi-manager farm; a literate manager/clerk exists; owner personally wants the control story (he is the buyer).
- [ ] Buying trigger noted (export coding? incident? new farm?) — shapes which module goes live first.
- [ ] Signed: ToS + onboarding scope + data-processing terms (see `LEGAL-*` drafts once ratified). Deposit collected.
- [ ] The 10-question interview (guide in `RESEARCH-customer-demand-2026-07-02.md` §f) — every onboarding doubles as research.

## Phase 1 — Ground-truth collection (on-farm day 1, ~4h)
- [ ] **Photograph every paper artifact**: attendance sheet, custody book + issue vouchers, spray log, harvest tally, the agronomist's season program. (These define the data model mapping AND the training vocabulary.)
- [ ] Structure walk: sectors/hawshat/lines counts, palm counts per hawsha (accept the farm's own numbers — reconcile later; never block on clean data, barrier #6).
- [ ] Store snapshot: item list + rough on-hand (the opening جرد happens in Phase 2).
- [ ] People list: staff roster + roles; crews stay head-count-level.
- [ ] Pick the farm's **go-live wedge** (one!): usually store control + purchase approvals (the anti-leakage story); export-driven farms may start with spray records.

## Phase 2 — System setup (remote, 1–2 days founder time)
- [ ] Org + roles created; owner/manager/accountant/storekeeper accounts (real emails; NO shared demo creds).
- [ ] Structure imported via the SPEC-0020 prefill/upsert path; Σ palms reconciled to the farm's number, variance documented not hidden.
- [ ] Item master + opening balances (from the opening count, entered as opening movements).
- [ ] Season program transcribed from the agronomist's table into templates (once template CRUD ships — OPS-PLAN L0-8; until then: plans authored directly).
- [ ] Org settings: weather thresholds, permissions verified per PERMISSIONS-MATRIX.

## Phase 3 — Training (on-farm day 2, half-day each role — the +20pp intervention)
- [ ] **Manager**: author a week (or confirm the dispatch screen when it ships), approve a PR, read the dashboards. Trained on HIS artifacts (the photos from Phase 1 → where each lives in the app).
- [ ] **Storekeeper**: receive stock, issue against ops, the جرد ritual (when the module ships). Framing: *the system protects YOU from accusation* (customer-research sabotage mitigations).
- [ ] **Accountant**: custody funding/settlement, expense entry, the monthly pack.
- [ ] **Owner** (30 min, remote OK): the control dashboard, approvals, what to check weekly.
- [ ] Leave-behind: 1-page Arabic quick cards per role (laminated), phone numbers.

## Phase 4 — Assisted operation (2 weeks, remote, ≤30 min/day founder time)
- [ ] Daily WhatsApp check-in with the manager (their channel, our cadence).
- [ ] Watch the data: entries/day per role (the adoption metric); silent role = call, don't wait.
- [ ] Fix data mistakes WITH the clerk on a call (teaching), never silently.
- [ ] End-of-week-2 review call with the owner: show his farm in his dashboard (the aha moment).

## Phase 5 — Hand-off (DFY→DIWY, the SaaS conversion)
- [ ] Named **farm clerk (كاتب المزرعة)** owns data entry going forward; certified on the 5 core flows.
- [ ] Founder-hours/farm/month target after hand-off: **≤2h** (measure it; if not decaying, STOP adding farms — GTM failure pattern #4).
- [ ] Subscription confirmed (harvest-aligned billing if chosen); renewal date = post-harvest with the season ledger in hand.
- [ ] Case-study consent asked (named before/after ledger — the GTM asset).

## Instrumentation (per farm, tracked in a simple sheet until multi-tenant admin exists)
onboarding start/go-live dates · founder-hours by phase · entries/day per role at wk 2/4/8 · wedge module chosen · incidents · renewal outcome.

## Known product gaps this playbook currently works around
Template CRUD (L0-8) · dispatch screen (L1-1) · stock-take module (جرد — inventory wave) · storekeeper `/m/receive` (REVIEW-360 F6) · per-org onboarding wizard (Tier-3 backlog — this playbook IS the manual version until then).
