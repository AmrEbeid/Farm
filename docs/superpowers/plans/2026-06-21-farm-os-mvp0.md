# Farm OS MVP-0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. The stock-coverage engine (Phase C) and every RLS policy (Phase B) are **checks-first** — the test is the oracle and is written and run-to-FAIL before any implementation. Each task ends with a commit. **Owner-gate every phase boundary; never auto-continue (PROJECT RULES).**

**Goal:** Prove the wedge loop end-to-end on the Ebeid reference tenant (`مزارع عبيد`) — *plan a month → see stock coverage → hit a budget gate → draft a purchase request → owner-approve → receive → record the field operation → see planned-vs-actual in the farm file* — against real seed tables (not mocks), passing the 09 acceptance gates (build spec §3, §10).

**Architecture:** Next.js App Router + TypeScript. RSC for reads (RLS-scoped Supabase queries), Server Actions / Route Handlers for writes. **Heavy logic lives in Postgres**: `fn_stock_coverage`, the reserve/issue movement posting, the budget check, and the bin materialization run as DB functions close to the data so the API and a future AI RPC reuse them. Tenancy is enforced in Postgres RLS (deny-by-default, `org_id` on every tenant table, membership join via `auth.user_org_ids()`), never only in the app layer. The append-only `farm_event` table with a `status` lifecycle (`planned → reserved → done`) is the spine the stock simulation iterates over — plan, reservation, and actual are the same row at different stages. `inventory_bin` is the materialized snapshot per item×location so reads never re-sum the ledger; `Σ(movements) == inventory_bin.on_hand` is a CI invariant. Presentation composes `@amrebeid/ui` (Sub-project A); this app owns all Arabic copy, `dir="rtl"`, auth, data, and workflows. PWA caches read shells and queues field-capture drafts in IndexedDB; approvals and financial/stock posting are online-only.

**Tech Stack:** Next.js (App Router) 14+ + TypeScript (`strict`); Supabase (Postgres 15 + PostGIS, Auth phone-OTP, Storage, RLS) via `@supabase/ssr`; Tailwind CSS (RTL: `dir="rtl"`, logical utilities); `@amrebeid/ui` (ThemeProvider + the 9 shipped components) + Tailwind for layout the library does not yet cover; Recharts for the PAB/variance charts; Vercel hosting; a PWA service worker (`next-pwa` or hand-rolled) for offline field-capture drafts. Tests: **Vitest** for the stock-engine pure-calc unit cases; **pgTAP** (preferred) or raw SQL assertions run via `supabase db execute` for RLS + DB invariants; **Playwright** for the end-to-end wedge loop.

> **Reality note on `@amrebeid/ui` (read before Phase D).** The library currently ships **9 components**: `Button`, `Field`, `Tag`, `KpiCard`, `Progress`, `Card`, `Alert`, `Tabs`, `VerdictBanner`, plus `ThemeProvider`/`useTheme`/`brandVars`. ## Global Constraints

Copied verbatim from the build spec hard rules (§2, §8, §9) and PROJECT RULES. These bind every task.

- **RLS deny-by-default + `org_id` isolation.** Every tenant table has an indexed `org_id`, RLS `ENABLE`d, policies `TO authenticated`, and joins membership via the security-definer helper `auth.user_org_ids()`. Tenancy is enforced in Postgres, **never only in the app layer**. A guessed org id returns zero rows (TI-1).
- **Separation of duties is absolute — author ≠ approver.** Only `owner` can approve a budget-breaching PR, enforced at the RLS/`authorize` layer **not the UI** (AP-1); the PR author cannot self-approve (AP-2); approvals are idempotent and reject stale versions (AP-3).
- **Audit-on-write immutability.** Every state change writes an `audit_log` row via AFTER trigger; `audit_log` has **no UPDATE and no DELETE policy → immutable by omission** (AP-4). Approved financial-style records (PR/budget commitments) are corrected via reversing entries, never silently edited (FC-1/FC-2).
- **Arabic-RTL-first.** RTL layout, فدان/قيراط, قطاع/حوشة/خط/نخلة; not an afterthought. The library ships no strings — **this app owns all Arabic copy, i18n, and `dir`**.
- **Offline only for field capture; online for approvals/financial posting.** Drafts (operation reports, issues, photos, stock requests) save to IndexedDB with a "pending sync" badge and sync on reconnect (OF-1). Approvals, final stock posting, financial posting, and permission changes are **online-only** and the UI says so when offline (OF-2). No full offline *sync* in MVP-0 — drafts only.
- **Performance targets.** Owner dashboard renders **< 2s** (PF-1); `fn_stock_coverage` returns **< 300ms** per item×location (PF-2); `farm_event` partitioned by month with BRIN on `occurred_at` + composite `(org_id, occurred_at)`; `inventory_bin` kept materialized (PF-3).
- **Never fabricate financial or stock numbers.** Use the real Ebeid registry/accounting figures from the seed (§9). If data is missing, say so — do not invent. The seed deliberately reproduces the real shortage (300 vs 500, 5-day lead) and budget breach.
- **Secrets server-side only.** No service-role key or secret in the client bundle. Supabase service-role client is server-only.
- **Commit after each task.** `git add` the exact paths, conventional-commit subject, body ending with:
  `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`
- **Independent review required** (PROJECT RULES) before merging any change to RLS/access, budget logic, the stock-coverage engine, or audit immutability. These are the High/Critical tasks below.

```sql
create table public.organization (
  id uuid primary key default gen_random_uuid(),
  name text not null, locale text not null default 'ar',
  currency text not null default 'EGP', area_unit text not null default 'feddan',
  fiscal_year_start date, settings jsonb not null default '{}'::jsonb
);
create table public.organization_member (
  org_id uuid not null references public.organization(id) on delete cascade,
  user_id uuid not null references auth.users(id) on delete cascade,
  role text not null check (role in
    ('owner','farm_manager','agri_engineer','accountant','storekeeper','supervisor')),
  scope jsonb not null default '{}'::jsonb,
  primary key (org_id, user_id)
);
create index on public.organization_member(user_id);

create or replace function auth.user_org_ids()
returns setof uuid language sql stable security definer set search_path = '' as $$
  select org_id from public.organization_member where user_id = (select auth.uid())
$$;

-- RBAC: role→permission map, called by policies as authorize('pr.approve')
create or replace function public.authorize(perm text)
returns boolean language sql stable security definer set search_path = '' as $$
  select exists (
    select 1 from public.organization_member m
    where m.user_id = (select auth.uid())
      and ( (perm = 'pr.approve' and m.role = 'owner')
         or (perm = 'plan.write'  and m.role in ('owner','farm_manager'))
         or (perm = 'op.execute'  and m.role in ('owner','farm_manager','agri_engineer','supervisor'))
         or (perm = 'inventory.write' and m.role in ('owner','farm_manager','storekeeper')) )
  )
$$;
alter table public.organization enable row level security;
alter table public.organization_member enable row level security;
create policy org_read on public.organization for select to authenticated
  using (id in (select auth.user_org_ids()));
create policy member_read on public.organization_member for select to authenticated
  using (org_id in (select auth.user_org_ids()));
```

```ts
export const available = (onHand: number, reserved: number, expired: number) =>
  onHand - reserved - expired;
export const safetyStock = (z: number, sigmaD: number, leadDays: number) =>
  z * sigmaD * Math.sqrt(leadDays);
export const reorderPoint = (dBar: number, leadDays: number, ss: number) =>
  dBar * leadDays + ss;
export function pabSeries(opening: number, issues: number[], receipts: number[]): number[] {
  const out = [opening];
  for (let t = 0; t < issues.length; t++)
    out.push(out[t] - (issues[t] ?? 0) + (receipts[t] ?? 0));
  return out;
}
export function firstShortagePeriod(series: number[], threshold = 0): number | null {
  for (let t = 1; t < series.length; t++) if (series[t] < threshold) return t;
  return null;
}
export const coverageDays = (avail: number, dailyDemand: number) =>
  dailyDemand <= 0 ? Infinity : avail / dailyDemand;
export function recommendation(
  { shortfall, ss, scheduledReceipts, packSize }:
  { shortfall: number; ss: number; scheduledReceipts: number; packSize: number }
) {
  const raw = Math.max(0, shortfall + ss - scheduledReceipts);
  const qty = Math.ceil(raw / packSize) * packSize;
  return {
    qty, orderByToday: qty > 0,
    message_ar: `⚠️ نقص متوقع: ${shortfall} كجم الأسبوع القادم. اطلب ${qty} كجم اليوم.`,
  };
}
```

```sql
create or replace function public.fn_stock_coverage(
  p_item uuid, p_location text default 'main', p_horizon int default 8)
returns jsonb language plpgsql stable security definer set search_path='' as $$
declare
  v_org uuid; v_onhand numeric; v_reserved numeric; v_expired numeric;
  v_ss numeric; v_lead int; v_pack numeric; v_avail numeric;
  v_issues numeric[]; v_receipts numeric[]; v_pab numeric[]; v_open numeric;
  v_first int := null; v_daily numeric; v_cov numeric; v_short numeric; v_qty numeric;
begin
  select org_id, on_hand, reserved into v_org, v_onhand, v_reserved
    from public.inventory_bin where item_id=p_item and location=p_location;
  if v_org not in (select auth.user_org_ids()) then raise exception 'forbidden'; end if;
  select coalesce(safety_stock,0), coalesce(lead_time_days,0), coalesce(pack_size,1)
    into v_ss, v_lead, v_pack from public.inventory_items where id=p_item;
  -- expired stock before the first planned consumption
  select coalesce(sum(qty),0) into v_expired from public.inventory_movements
    where item_id=p_item and type='expiry';
  v_avail := v_onhand - v_reserved - v_expired;
  -- planned issues per period from reserved/planned consumption events (negative inventory_adjustment)
  select array_agg(s.iss order by s.t) into v_issues from (
    select width_bucket(extract(epoch from q_period)::int,0, p_horizon*7*86400, p_horizon) t,
           sum(-q.inventory_adjustment) iss
    from public.quantities q
    join public.farm_event e on e.id=q.event_id and e.status in ('planned','reserved')
    , lateral (select extract(epoch from (e.occurred_at - now()))) as q_period
    where q.material_id=p_item and q.inventory_adjustment < 0
    group by 1) s;
  v_open := v_avail;
  v_pab := array[v_open];
  v_short := 0;
  for i in 1 .. coalesce(array_length(v_issues,1),0) loop
    v_pab := v_pab || (v_pab[i] - coalesce(v_issues[i],0));      -- no scheduled receipts in MVP-0 base case
    if v_first is null and v_pab[i+1] < 0 then v_first := i; v_short := -v_pab[i+1]; end if;
  end loop;
  v_daily := coalesce(v_issues[1],0) / 7.0;
  v_cov := case when v_daily<=0 then 'infinity'::numeric else v_avail / v_daily end;
  v_qty := ceil(greatest(0, v_short + v_ss) / v_pack) * v_pack;
  return jsonb_build_object(
    'available', v_avail,
    'reorder_point', (coalesce(v_issues[1],0)) + v_ss,
    'coverage_days', round(v_cov,1),
    'stockout_date', (now() + (v_cov || ' days')::interval),
    'pab', to_jsonb(v_pab),
    'first_shortage_period', v_first,
    'recommend_qty', v_qty,
    'order_by', case when v_cov < v_lead then current_date else null end,
    'message_ar', format('⚠️ نقص متوقع: %s كجم. الغطاء %s يوم < مهلة %s يوم. اطلب %s كجم اليوم.',
                         v_short, round(v_cov,1), v_lead, v_qty));
end $$;
```
