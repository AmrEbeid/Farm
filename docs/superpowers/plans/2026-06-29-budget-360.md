# Budget 360 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task. Keep the checklist current while working.

**Goal:** Add a read-only Budget 360 route at `/budgets/[budgetId]` and link budget list rows to it.

**Architecture:** Build an additive read-only Server Component from existing RLS-scoped `budgets`, `budget_lines`, `expenses`, and `purchase_requests` tables. No migrations, no RPCs, no writes, no accounting/P&L draft work, no package UI changes.

## Constraints

- Arabic RTL first.
- No fabricated financial data; all KPIs must be query-derived.
- No migrations, RPC changes, service-role usage, or writes.
- Do not use draft accounting/P&L work that is not on `main`.
- Do not change `packages/ui`.
- Do not import chart libraries.
- Validate with targeted lint, `npx tsc --noEmit`, `npx vitest run`, and `npm run build` from `apps/farm-os`.
- Update docs last.
- Do not merge, migrate, deploy, or apply database changes.

## Budget 360 Contract

- Header/profile: name, period, category, status.
- KPI cards: planned, approved, committed+actual, available.
- Tables: budget lines, recent expenses for the same category, purchase requests linked to the budget id when present.
