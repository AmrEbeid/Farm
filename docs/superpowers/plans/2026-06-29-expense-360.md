# Expense 360 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement this plan task-by-task. Keep the checklist current while working.

**Goal:** Add a read-only Expense 360 route at `/expenses/[expenseId]` and link expense list rows to it.

**Architecture:** Build an additive read-only Server Component from existing RLS-scoped `expenses` and its existing references to supplier, plan, farm/sector/hawsha, and event. No migrations, no RPCs, no writes, no draft accounting/P&L work, no package UI changes.

## Constraints

- Arabic RTL first.
- No fabricated financial data; all KPIs/details must be query-derived.
- No migrations, RPC changes, service-role usage, or writes.
- Do not use draft accounting/P&L work that is not on `main`.
- Do not change `packages/ui`.
- Do not import chart libraries.
- Validate with targeted lint, `npx tsc --noEmit`, `npx vitest run`, and `npm run build` from `apps/farm-os`.
- Update docs last.
- Do not merge, migrate, deploy, or apply database changes.

## Expense 360 Contract

- Header/profile: category, date, description, status.
- KPI cards: total, quantity, unit price, linked scope count.
- Details: supplier, payment method, plan link, farm/sector/hawsha links, event link when present.
- Related navigation: back to Finance dashboard and Expenses list.
